## 🔎 Задание 3: Поиск + добавление поля `label` (filter + map) — Уровень JUNIOR

### Таблица методов

| Метод               | Тип          | Параметры                  | Возвращает   | Назначение                               |
| ------------------- | ------------ | -------------------------- | ------------ | ---------------------------------------- |
| `products.filter()` | Array Method | `(p) => условие поиска`    | Новый массив | Отбор товаров по названию                |
| `mapped.map()`      | Array Method | `(p) => ({ ...p, label })` | Новый массив | Добавление поля `label` к каждому товару |

### Почему используется `map`

Использую `map`, потому что нужно преобразовать каждый объект — добавить новое свойство `label`, сохранив все старые.
`map` возвращает массив той же длины, но с изменёнными данными.

```
// map: создаю новый массив товаров, добавляя каждому поле label с иконкой ⭐
// Использую map, потому что нужно преобразовать каждый объект — добавить
// новое свойство, сохранив все старые. map возвращает массив той же длины.
const mapped = filtered.map((p) => ({
    ...p,
    label: `⭐ Доступно: ${p.name}`,
}));

```

### 🛒 Задание 4: Корзина (forEach + map + reduce) — Уровень JUNIOR

| Функция / Метод        | Тип                  | Параметры                     | Возвращает       | Назначение                                |
| ---------------------- | -------------------- | ----------------------------- | ---------------- | ----------------------------------------- |
| `addToCart(productId)` | Function Declaration | `productId` (number)          | —                | Добавление товара в корзину с валидацией  |
| `renderCart()`         | Function Declaration | —                             | —                | Отрисовка корзины и подсчёт суммы         |
| `cart.forEach()`       | Array Method         | `(item) => {...}`             | —                | Вывод каждого товара корзины на страницу  |
| `cart.map()`           | Array Method         | `(item) => item.price`        | Array of numbers | Получение массива цен из объектов корзины |
| `prices.reduce()`      | Array Method         | `(sum, price) => sum + price` | number           | Подсчёт итоговой суммы                    |

> 💡 **Почему forEach**: Использую `forEach`, потому что мне нужно для каждого товара создать HTML-элемент и добавить его в блок корзины. Новый массив не нужен — я просто выполняю действие для каждого элемента.

> 💡 **Почему map + reduce**: Использую эту цепочку, потому что задание требует явно показать работу обоих методов. `map` трансформирует массив объектов в массив чисел (цен), а `reduce` агрегирует (суммирует) все цены в одно итоговое значение.

```javascript
// forEach: перебираю товары в корзине, чтобы показать их на странице
// Использую forEach, потому что мне нужно для каждого товара создать
// HTML-элемент и добавить его в блок корзины. Новый массив не нужен.
cart.forEach((item) => {
  const div = document.createElement("div");
  div.className = "cart-item";
  div.innerHTML = `<span>${item.name}</span> <span>${item.price} BYN</span>`;
  cartContainer.appendChild(div);
});

// map + reduce: подсчитываю итоговую сумму корзины
// map: беру каждый товар и оставляю только его цену → получаю массив чисел [2500, 45, 180]
// reduce: беру все цены и складываю их в одно число → получаю итоговую сумму
// Использую эту цепочку, потому что задание требует показать работу обоих методов.
const prices = cart.map((item) => item.price);
const total = prices.reduce((sum, price) => sum + price, 0);
cartTotal.textContent = `Итого: ${total.toFixed(2)} BYN`;
```

## ❓ Ответы на контрольные вопросы (Interview Questions)

### 1. В чём разница между `map()` и `forEach()`? Когда использовать каждый?

| Критерий                     | `map()`                                    | `forEach()`                                         |
| :--------------------------- | :----------------------------------------- | :-------------------------------------------------- |
| **Возвращает**               | Новый массив с преобразованными элементами | `undefined` (ничего не возвращает)                  |
| **Изменяет исходный массив** | Нет                                        | Нет                                                 |
| **Когда использовать**       | Когда нужно получить новый массив данных   | Когда нужно выполнить действие для каждого элемента |

**Пример из кода:**

```javascript
// map: получаю массив цен для подсчёта суммы
const prices = cart.map((item) => item.price); // [2500, 45, 180]

// forEach: вывожу каждый товар корзины на страницу
cart.forEach((item) => {
  const div = document.createElement("div");
  cartContainer.appendChild(div);
});
```

### 2. Что возвращает filter()? Изменяет ли он исходный массив?

> filter() возвращает новый массив с элементами, которые прошли проверку условия. Исходный массив не изменяется — это важно для сохранения целостности данных.
> Пример из кода:

```javascript
// filter: создаю новый массив с товарами из категории "Электроника"
// Исходный массив products остаётся без изменений
const filtered = products.filter((p) => p.category === "Электроника");
```

### 3. Что такое JSON? Чем JSON-строка отличается от JavaScript-объекта?

| Характеристика           | JSON-строка                   | JavaScript-объект              |
| :----------------------- | :---------------------------- | :----------------------------- |
| **Тип данных**           | Строка (text)                 | Объект (object)                |
| **Кавычки ключей**       | Только двойные `{"key": "v"}` | Можно без кавычек `{key: "v"}` |
| **Поддерживает функции** | ❌ Нет                        | ✅ Да                          |
| **Использование**        | Хранение и передача данных    | Логика внутри JS               |

**Пример из кода:**

```javascript
// Объект JS → JSON-строка (для localStorage)
const cartJSON = JSON.stringify(cart);

// JSON-строка → Объект JS (после загрузки)
const cart = JSON.parse(saved);
```

### 4. Что произойдёт, если передать в `JSON.parse()` невалидную строку?

> Будет выброшена ошибка **`SyntaxError`**. Чтобы избежать поломки приложения, рекомендуется использовать проверку или конструкцию `try/catch`.

**Пример из кода:**

```javascript
const saved = localStorage.getItem("myCart");
if (saved) {
  // Безопасный парсинг только при наличии данных
  cart = JSON.parse(saved);
} else {
  console.log("Данные не найдены");
}
```

### 5. Что такое деструктуризация объекта? Приведите пример из своего кода.

Деструктуризация — способ извлечь свойства объекта в отдельные переменные в одной строке.
Пример из кода:

```javascript
// Вместо:
const name = product.name;
const price = product.price;
const category = product.category;

// Используем деструктуризацию:
const { name, price, category } = product;
```

### 6. Что делает оператор spread { ... } при копировании объекта?

Spread-оператор создаёт поверхностную копию объекта, копируя все свойства в новый объект.
Пример из кода:

```javascript
// map: добавляю поле label к каждому товару
const mapped = filtered.map((p) => ({
  ...p, // копируем все старые свойства
  label: `⭐ ${p.name}`, // добавляем новое свойство
}));
```

### 7. Почему нельзя сравнивать объекты через ===? Что выведет: { } === { }?

Объекты сравниваются по ссылке, а не по значению. Два разных объекта с одинаковыми свойствами — это разные ссылки в памяти.
Ответ: { } === { } выведет false.
Пример из кода:

```javascript
// Сравниваем по id (число), а не по объекту
const product = products.find((p) => p.id === productId);
```

### 8. Что такое метод reduce()? Как вы использовали его для подсчёта суммы?

reduce() — «сворачивает» массив в одно значение (accumulator). Принимает функцию-колбэк и начальное значение.
Пример из кода:

```javascript
// map + reduce: подсчёт итоговой суммы корзины
const prices = cart.map((item) => item.price); // [2500, 45, 180]
const total = prices.reduce((sum, price) => sum + price, 0); // 2725

// sum — накопитель (начинается с 0)
// price — текущий элемент массива
// Результат: одно число (итоговая сумма)
```
